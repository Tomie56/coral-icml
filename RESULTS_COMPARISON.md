## 3. 🔍 差异根本原因分析

### 3.1 为什么实际实验的 Baseline 远低于论文？

**关键发现：实验使用了不同的评估协议**

#### 论文 Table 1 的 "Fewshot" baseline：
- 准确率：**50.84%**
- 使用 **lm-eval harness** 5-shot 评估
- 这是标准的 MMLU 评估方式

#### 实际实验的 baseline：
- 准确率：**36.93%**  
- 从 `metrics.json` 的 "baseline" 字段来看
- **可能使用了不同的提示格式或评估方式**

### 3.2 为什么实际实验的 CORAL 结果与论文接近？

虽然 baseline 差异巨大，但 CORAL 引导后的结果却很接近：
- 论文 CORAL：**53.97%**
- 实际 CORAL：**56.91%**
- 差异仅 **2.94%**

**这表明**：CORAL 引导机制能够"弥补"不同评估协议导致的 baseline 差异，使最终性能趋于一致。

### 3.3 最可能的解释

**实验可能使用了不同的数据子集或评估方式：**

1. **数据子集差异**：
   - 论文使用完整的 MMLU test set（所有57个科目）
   - 实际实验可能只使用了部分科目或样本
   - 从 `probe_data.npz` 的 33,700 训练样本看，这不是完整 MMLU

2. **评估协议差异**：
   - 论文使用 lm-eval harness 标准协议（5-shot）
   - 实际实验的 baseline 评估方式未知
   - 可能是 0-shot 或不同的提示模板

3. **样本选择偏差**：
   - 如果实际实验选择了模型表现较差的科目子集
   - Baseline 会更低，但 CORAL 的绝对提升会更大

## 5. 主要发现总结

### 5.1 数值对比表

| 指标 | 论文 Table 1 | 实际实验 | 差异 |
|------|-------------|---------|------|
| **Baseline 准确率** | 50.84% | 36.93% | -13.91% ⚠️ |
| **CORAL 准确率** | 53.97% | 56.91% | +2.94% |
| **准确率绝对提升** | +3.13% | +20.0% | +16.87% 🚨 |
| **准确率相对提升** | +6.2% | +54.1% | +47.9% 🚨 |
| **Baseline ECE** | 22.12 | 29.94 | +7.82 ⚠️ |
| **CORAL ECE** | 6.21 | 2.74 | -3.47 ✅ |
| **ECE 相对改进** | -71.9% | -90.9% | -19.0% |
| **训练样本数** | 8.4k-10k | 33.7k | +23.7k ~ +25.3k |

### 5.2 核心结论

1. **❌ Baseline 性能不匹配**：实际实验的 baseline（36.93%）远低于论文（50.84%），说明使用了不同的评估协议或数据子集

2. **✅ CORAL 最终性能接近**：尽管 baseline 差异巨大，CORAL 引导后的准确率（56.91% vs 53.97%）相差不大

3. **🚨 改进幅度被夸大**：由于 baseline 更低，实际实验显示的改进幅度（+20%）远超论文报告（+3.13%），但这是**误导性的**

4. **📊 使用了更多训练数据**：实际实验使用 33.7k 训练样本，是论文声称的 3-4 倍

## 6. ⚠️ 重要结论与建议

### 6.1 实验配置不匹配论文

**实际实验与论文 Table 1 报告的不是同一个实验**，主要差异：

| 维度 | 论文 Table 1 | 实际实验 |
|------|-------------|---------|
| Baseline 方法 | lm-eval 5-shot (50.84%) | 未知方法 (36.93%) |
| 数据集 | 完整 MMLU test | 可能是子集 |
| 训练样本 | 8.4k-10k | 33.7k |
| Baseline ECE | 22.12 | 29.94 |

### 6.2 不能直接对比改进幅度

**❌ 错误的对比方式**：
- "实际实验准确率提升 +20%，远超论文的 +3.13%"

**✅ 正确的理解**：
- 由于 baseline 评估方式不同，改进幅度不可比
- 应该对比 **CORAL 最终性能**：论文 53.97% vs 实际 56.91%（接近）
- 实际实验的 +20% 提升是在**更低 baseline**上的结果

### 6.3 推荐的报告方式

如果要报告这个实验结果，应该：

#### ✅ 正确做法：
```markdown
**DeepSeek-7B-Chat on MMLU (本实验配置):**
- Baseline: 36.93% accuracy, 29.94 ECE
- CORAL: 56.91% accuracy, 2.74 ECE
- 改进: +20.0% accuracy, -90.9% ECE

注：本实验的 baseline 评估协议与论文 Table 1 不同，
因此改进幅度不可直接对比。CORAL 最终性能（56.91%）
与论文报告（53.97%）接近。
```

#### ❌ 错误做法：
```markdown
"实验结果显示 CORAL 在 MMLU 上提升了 20% 准确率，
远超论文报告的 3% 提升"  
← 这是误导性的，因为 baseline 不同
```

### 6.4 需要进一步调查

为了准确对比，需要：

1. **确认实际实验的 baseline 是如何计算的**
   - 查看 `mlp_steer_mmlu.py` 的评估逻辑
   - 确认是否使用了 lm-eval harness 协议

2. **确认使用的 MMLU 数据范围**
   - 是完整 57 个科目还是子集？
   - 训练和测试样本的具体来源

3. **重新运行实验以匹配论文设置**
   - 使用标准 lm-eval 5-shot 评估
   - 使用完整 MMLU test set
   - 使用论文规模的训练数据（8-10k）
